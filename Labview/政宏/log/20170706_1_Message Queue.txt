Support>Message Queue
-----
    使用的應該是Labview內部的subvi再加上自己的一些修改

Obtain Message Queue
    Labview原生的QMH Project中的元件，產生Cluster(Message+MessageData)的Queue Controls

Create All Message Queues
    Labview原生的QMH Project中的元件，直接把"Initialize" Enqueue進去"Obtain Message Queue" Obtain 出來的Queue Controls。
    再將所有的Queue Controls Bundle起來變成由Queue Control 組成的Message Queues Out
    比較不同的是在此處可由使用者設定Message Queues(設定為TYPE DEF)的項目。
    原生的Create All Message Queues只有UI，此處還增加了Task、Moter、DAQ、Logging。

Enqueue Message
    Labview原生的QMH Project中的元件，
    這是一個polymorphic的SubVI，包含了Enqueue Message (Single)和Enqueue Message (Array)。

Enqueue Message (Single)
    Labview原生的QMH Project中的元件，
    顧名思義，就是Enqueue Single Message 

Enqueue Message (Array)
    Labview原生的QMH Project中的元件，
    顧名思義，就是Enqueue Array Message 

Dequeue Message
    Labview原生的QMH Project中的元件，
    顧名思義，就是Enqueue Array Message 

Message Queue.lvlib
    就是個lvlib，包含所有東西。

All Message Queues.ctl
    定義了Message Queues Cluster中，有多少個Queue元素，
    此處包含了UI、Task、Moter、DAQ、Logging。

Message Cluster.ctl
    QMH架構中Element的定義，Message Cluster{Message(string)、Message Data(variant)}


---------------------
此處沒有太多的東西，
唯一要注意的是All Message Queues.ctl和Create All Message Queues會隨案件不同而改變。
但是也有可能把所有可能用的到的東西包進去，固定住。
